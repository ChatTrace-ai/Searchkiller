二、ADC 认证配置

核心原则：不需要 API Key。本项目通过 @ai-sdk/google-vertex 使用 Application Default Credentials (ADC)。本地和 Cloud Run 上代码完全一致，零密钥管理。

本地开发环境配置

获取临时本地凭据 
gcloud auth application-default login

这会在 ~/.config/gcloud/application_default_credentials.json 创建临时凭据文件。SDK 自动读取此文件，无需手动指定路径。

配置 .env 文件 
# .env — 只需项目 ID 和区域，不需要 API Key
GOOGLE_VERTEX_PROJECT=your-gcp-project-id
GOOGLE_VERTEX_LOCATION=us-central1

# 其他服务
EXA_API_KEY=your-exa-key
ES_CLOUD_ID=your-es-cloud-id
ES_API_KEY=your-es-api-key
NEXT_PUBLIC_APP_URL=http://localhost:3000
验证配置 
# 测试 ADC 是否生效
gcloud ai endpoints list --region=us-central1 \
  --project=your-gcp-project-id

# 或直接测试 API
npm run dev
curl -X POST http://localhost:3000/api/plan \
  -H "Content-Type: application/json" \
  -d '{"keyword": "test query"}'
优势：
  • 绕过"禁止创建 API 密钥"的组织策略限制
  • 无长期有效密钥字符串泄露风险
  • 本地与 Cloud Run 代码行为完全一致

Cloud Run 生产环境

创建专用服务账号 
gcloud iam service-accounts create g-rapid-agent-sa \
  --display-name="G-RapidAgent Cloud Run SA" \
  --project=YOUR_PROJECT_ID
授予最小权限 
SA=g-rapid-agent-sa@YOUR_PROJECT_ID.iam.gserviceaccount.com

# Vertex AI 调用权限
gcloud projects add-iam-policy-binding YOUR_PROJECT_ID \
  --member="serviceAccount:$SA" \
  --role="roles/aiplatform.user"

# Secret Manager 读取权限
gcloud projects add-iam-policy-binding YOUR_PROJECT_ID \
  --member="serviceAccount:$SA" \
  --role="roles/secretmanager.secretAccessor"

# 日志和监控
gcloud projects add-iam-policy-binding YOUR_PROJECT_ID \
  --member="serviceAccount:$SA" \
  --role="roles/logging.logWriter"
gcloud projects add-iam-policy-binding YOUR_PROJECT_ID \
  --member="serviceAccount:$SA" \
  --role="roles/monitoring.metricWriter"
部署时指定服务账号 
gcloud run deploy g-rapid-agent \
  --service-account=g-rapid-agent-sa@YOUR_PROJECT_ID.iam.gserviceaccount.com \
  --set-env-vars="GOOGLE_CLOUD_PROJECT=YOUR_PROJECT_ID,GOOGLE_CLOUD_REGION=us-central1" \
  ...

Cloud Run 容器内的 google-auth-library 会自动从 metadata server 获取服务账号凭据。

代码实现 — lib/gemini.ts

import { createVertex } from '@ai-sdk/google-vertex';

// SDK 自动从 ADC 获取凭据:
//   本地: ~/.config/gcloud/application_default_credentials.json
//   Cloud Run: metadata server (服务账号)
const vertex = createVertex({
  project: process.env.GOOGLE_VERTEX_PROJECT,
  location: process.env.GOOGLE_VERTEX_LOCATION,
});

export const flashModel = vertex('gemini-2.5-flash');
export const proModel = vertex('gemini-2.5-pro');
注意：不要使用 GOOGLE_APPLICATION_CREDENTIALS 环境变量指向 JSON Key 文件。ADC 自动检测机制更安全，且与 Cloud Run 兼容。

四、一键部署流程

使用 scripts/setup-gcp.sh 一键完成从零到部署。
前置条件 
# 安装 gcloud CLI
curl https://sdk.cloud.google.com | bash
gcloud init
gcloud auth login
执行一键部署 
bash scripts/setup-gcp.sh YOUR_PROJECT_ID

脚本自动完成 7 步：

  1. 开启 Cloud Run / Artifact Registry / Vertex AI / Cloud Build / Secret Manager API
  2. 创建专用服务账号 + 最小 IAM 权限
  3. 创建 Artifact Registry 仓库
  4. 配置 Secret Manager (交互式输入 Exa/ES 密钥)
  5. Cloud Build 构建 Docker 镜像
  6. 部署到 Cloud Run
  7. Smoke Test (健康检查 + Plan API 测试)
后续更新 
# 代码更新后快速重新部署
gcloud builds submit . \
  --tag us-central1-docker.pkg.dev/YOUR_PROJECT_ID/g-rapid-agent/app:latest
gcloud run deploy g-rapid-agent \
  --image us-central1-docker.pkg.dev/YOUR_PROJECT_ID/g-rapid-agent/app:latest \
  --region us-central1

五、配额与限流指南

Vertex AI DSQ (Dynamic Shared Quota) 分层

模型Tier 1 ($10-$250/月)Tier 2 ($250-$2K)Tier 3 (>$2K)系统限制
Gemini Flash2M TPM4M TPM10M TPM30K RPM/区域
Gemini Pro500K TPM1M TPM2M TPM10 QPM 硬限制
关键限制：Gemini 2.5 Pro 有 10 QPM (每分钟 10 次查询) 的硬限制，不随 Tier 升级。多用户并发时 Pro 会成为瓶颈。考虑：
  • 将非关键 Pro 调用降级为 Flash
  • 购买 Provisioned Throughput 获取保证吞吐
  • 使用 global endpoint 而非区域端点 (GOOGLE_VERTEX_LOCATION=global)

六、故障排除

▶ 429 RESOURCE_EXHAUSTED — Gemini 限流

原因：DSQ 共享池容量不足或 Pro 超过 10 QPM。

解决方案：

  1. gemini-client.ts 自动重试 (已内置, 最多 3 次指数退避)
  2. 切换到 global endpoint: GOOGLE_VERTEX_LOCATION=global
  3. 增加消费以升级 Tier (30 天自动评估)
  4. 购买 Provisioned Throughput
▶ 403 PERMISSION_DENIED — 权限不足

本地开发：

gcloud auth application-default login
gcloud ai endpoints list --region=us-central1 --project=YOUR_PROJECT

Cloud Run：

gcloud projects add-iam-policy-binding YOUR_PROJECT \
  --member="serviceAccount:SA_EMAIL" \
  --role="roles/aiplatform.user"
▶ Cloud Run 503 / 超时

超时：Cloud Run 默认 300s。Harness 多轮循环可能超时。

# 增加超时
gcloud run services update g-rapid-agent \
  --timeout=600 --region=us-central1

# 检查日志
gcloud run services logs read g-rapid-agent --region=us-central1

冷启动：确保 --min-instances=1。

▶ Elasticsearch 连接失败

确认 ES_CLOUD_ID 和 ES_API_KEY 正确。在 Secret Manager 中检查：

gcloud secrets versions access latest --secret=es-cloud-id
gcloud secrets versions access latest --secret=es-api-key
