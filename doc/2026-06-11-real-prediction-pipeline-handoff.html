G-RapidAgent 真实数据预测管线 — 交接文档

日期：2026-06-11  |  项目：G-RapidAgent / Searchkiller

本文档介绍从 mock 预设数据 迁移到 Exa + Gemini 真实数据管线 的完整改造。 预测结果现由 Exa.ai 语义搜索与 Google Gemini 多阶段推理生成，Elasticsearch 持久化存储，前端通过轮询获取异步结果。

概述

此前，预测功能依赖 prediction-seeds.ts 中的 mock 预设数据和定时器模拟延迟， 无法反映真实市场信号与语义检索能力。本次改造引入独立的 lib/prediction-generator.ts 管线，串联 Gemini Flash（规划与分析）、 Exa.ai（语义搜索）与 Gemini Pro（报告生成），实现端到端真实预测生成。

prediction-store.ts 负责 API 层编排：创建 ES skeleton、触发 fire-and-forget 异步生成、 并在完成后更新文档状态。前端无需改动轮询协议，只需等待更长的真实生成时间（约 60–90 秒）。

架构图

以下流程展示一次完整预测请求的生命周期：

↓

↓

Exa.ai → Semantic search (6–12 sources)
Gemini Flash → Analyze probabilities
Gemini Pro → Generate report (streaming)
ES → Update to completed
↓
前端 → 轮询 GET /api/predictions/:id → 获取真实结果 
# 等效伪代码
POST /api/predictions  →  create skeleton (processing)
void generateRealPrediction(id, query)  // fire-and-forget
  subQueries = geminiFlash.plan(query, n=4)
  sources    = exa.search(subQueries, limit=6-12)
  probs      = geminiFlash.analyze(sources)
  report     = geminiPro.streamReport(probs, sources)
  elasticsearch.update(id, { status: "completed", ...report })

API 端点说明

POST /api/predictions

创建新预测。写入 ES skeleton（status: processing），并 fire-and-forget 触发真实生成管线。

GET /api/predictions/:id

获取预测详情。前端轮询此端点，直到 status 从 processing 变为 completed。

POST /api/predictions/:id/refresh

刷新已有预测。重新执行完整生成管线，覆盖旧结果。

GET /api/predictions/popular

获取热门预测列表。用于首页 featured 展示。

# 典型调用流程
curl -X POST /api/predictions -d '{"query":"Will X happen by 2026?"}'
# → { "id": "abc123", "status": "processing" }

while status == "processing":
  curl /api/predictions/abc123  # 每 3–5 秒轮询
# → status: "completed", report: {...}
