预测 API v1

帮助前端和后端开发者快速理解当前预测生命周期、请求响应契约、游标分页、轮询方式以及 Mock 行为。

基础路径/api

当前已经实现什么

当前接口已经可以直接供前端使用，但数据存储和预测生成暂时由进程内存模拟。后续替换为 Elasticsearch 和真实模型生成时，请求与响应结构应保持不变。

32 条热门预测

共两页，每页 16 条，用于首页开发和演示。

2 秒模拟生成

新建或刷新的预测先进入 processing，随后自动变成 completed。

进程内临时状态

Next.js 进程重启后数据会重置，当前不能作为持久化数据库使用。

前端请求流程

首页读取预测摘要；用户提交问题后创建或复用预测；详情页持续轮询，直到结果生成完成。

1 · 加载首页GET 热门预测，每页 16 条
2 · 提交问题POST 用户想预测的问题
3 · 页面跳转打开 /prediction/{id}
4 · 轮询详情按照 pollAfterMs 请求详情，直到 completed

核心数据模型

PredictionSummary

用于首页卡片的轻量数据。

{
  "id": "world-cup-2026",
  "question": "Who will win the 2026 FIFA World Cup?",
  "category": "Sports",
  "icon": "trophy",
  "status": "completed",
  "topOutcomes": [
    { "label": "Spain", "probability": 16.4, "icon": "🇪🇸" }
  ],
  "updatedAt": "2026-06-11T08:00:00.000Z"
}

PredictionDetail

详情页渲染所需的完整预测结果。

{
  "id": "world-cup-2026",
  "status": "completed",
  "confidence": {
    "level": "high",
    "score": 82,
    "explanation": "Stable signals..."
  },
  "outcomes": [],
  "sources": [],
  "summary": [],
  "report": "## Forecast analysis...",
  "createdAt": "...",
  "updatedAt": "..."
}
概率约束：详情中的所有结果概率总和必须为 100，最多允许 ±0.1 的舍入误差。未列出的概率由 “Other outcomes” 补齐。

热门预测列表

GET/api/predictions/popular首页

返回热门预测摘要

使用不透明游标分页。前端必须原样回传接口返回的 cursor，不要自行解析或拼接。

参数必填规则
cursor否第一页不传；后续原样回传 nextCursor
limit否默认 16；必须是 1–32 的整数
category否分类精确匹配，不区分大小写

200 返回 items、nextCursor 和 hasMore。

400 limit 或 cursor 无效。

请求示例 
GET /api/predictions/popular?limit=16
响应示例 
{
  "items": [PredictionSummary],
  "nextCursor": "eyJvZmZzZXQiOjE2fQ",
  "hasMore": true
}

创建或复用预测

POST/api/predictions用户操作

接收自然语言预测问题

服务端会去除首尾空格，问题长度必须为 5–500 个字符。复用匹配时忽略大小写和标点差异。

processing→ 模拟生成 2 秒 →completed

200 命中已有 completed 结果，直接复用。

202 已创建新的 Mock 生成任务。

400 JSON 格式错误或问题长度不合法。

请求体 
{
  "question": "Who will win the 2026 FIFA World Cup?"
}
新建预测响应 
{
  "id": "pred_01JXYZ",
  "status": "processing",
  "reused": false,
  "pollAfterMs": 2000
}

预测详情

GET/api/predictions/{id}轮询接口

返回生成进度或完整详情

processing 和 completed 都返回 HTTP 200。前端必须根据 JSON 中的 status 字段判断状态，不能只看 HTTP 状态码。

状态前端处理方式
processing展示进度，并在 2 秒后再次请求
completed渲染置信度、候选结果、来源、摘要和报告
failed预留状态；展示可恢复的失败页面
处理中响应 
{
  "id": "pred_01JXYZ",
  "question": "...",
  "status": "processing",
  "progress": {
    "stage": "collecting_sources",
    "message": "Collecting reference sources"
  },
  "updatedAt": "..."
}

刷新预测

POST/api/predictions/{id}/refresh可选功能

重新启动预测生成

completed 预测会重新进入 2 秒的 processing 状态。热门预测仍永久有效；自定义预测会重新获得 24 小时的 Mock 有效期。

202 已开始刷新。

409 当前预测正在生成，不能重复刷新。

404 预测 ID 不存在。

响应示例 
{
  "id": "world-cup-2026",
  "status": "processing",
  "pollAfterMs": 2000
}

统一错误格式

响应结构

{
  "error": {
    "code": "PREDICTION_NOT_FOUND",
    "message": "Prediction was not found."
  }
}

错误码

400INVALID_REQUEST
404PREDICTION_NOT_FOUND
409PREDICTION_IN_PROGRESS
429RATE_LIMITED（预留）
500GENERATION_FAILED（预留）
503SERVICE_UNAVAILABLE（预留）

前后端接入指南

前端职责

不要解析 cursor；每页展示 16 张卡片；使用返回的 ID 跳转；按照 pollAfterMs 轮询；根据 status 分支处理；向用户展示 API 错误信息。

const created = await fetch("/api/predictions", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ question })
}).then(r => r.json());

router.push(`/prediction/${created.id}`);

后端替换约束

将进程内存替换为 Elasticsearch 时，不要改变接口请求和响应结构。需要持久化标准化问题、状态、详情、过期时间和生成状态。

// 以下接口保持稳定：
GET  /predictions/popular
POST /predictions
GET  /predictions/:id
POST /predictions/:id/refresh
